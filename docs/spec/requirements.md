# 要件定義書 — 汎用列車制御装置

**バージョン**: 1.0.0
**作成日**: 2026-06-01
**対象プロジェクト**: stormworks_train_controller

---

## 1. プロジェクト概要

Stormworks: Build and Rescue のマイコン（Lua 5.3 サンドボックス）上で動作する、**汎用列車制御装置**を開発する。

### 1.1 対象車両種別

| 識別子 | 説明 | プロパティ |
|--------|------|------------|
| TA | 先頭車（前部運転台） | `Is TA = true` |
| TB | 中間車（客車・運転台なし） | `Is TA = false`, `Is TAB = false` |
| TAB | 最後尾車（後部運転台） | `Is TAB = true` |
| 一両車 | 先頭・最後尾を兼ねる単体車両 | `Is TA = true`, `Is TAB = true` |

---

## 2. 出力チャンネル仕様

各マイコンはゲーム内アクチュエータへ以下の信号を出力する。

| 機能 | 型 | TA | TB | TAB |
|------|----|----|----|-----|
| スロットル | Number (0〜1) | ✓ | ✓ | ✓ |
| ブレーキ | Number (0〜1) | ✓ | ✓ | ✓ |
| 右ドア | Bool | ✓ | ✓ | ✓ |
| 左ドア | Bool | ✓ | ✓ | ✓ |
| 室内ライト | Bool | ✓ | ✓ | ✓ |
| スポットライト | Bool | ✓ | — | ✓ |

---

## 3. 車間通信プロトコル

### 3.1 通信原則

- 各マイコンは**隣接する車両とのみ**コンポジット信号で通信する。
- 前方コネクタと後方コネクタは**信号内容で識別**する。
- 車両の向きは、どちら側のコネクタから「前方マーカー信号」が来るかで判断する。

### 3.2 車間コンポジット信号チャンネル定義

各コネクタに対して以下のチャンネルを使用する（双方向同一定義）。

**Number チャンネル**

| CH | 用途 | 値域 |
|----|------|------|
| 1 | スロットル命令 | 0.0〜1.0 |
| 2 | ブレーキ命令 | 0.0〜1.0 |
| 3 | コネクタ識別マーカー（前方=1.0 / 後方=0.0） | 0.0 or 1.0 |

**Bool チャンネル**

| CH | 用途 |
|----|------|
| 1 | マスター存在フラグ（自車または前方にマスターがいる） |
| 2 | マスター要求信号（TAまたはTABがリクエスト送信） |
| 3 | マスター方向（true = TA方向が前 / false = TAB方向が前） |
| 4 | 右ドア命令 |
| 5 | 左ドア命令 |
| 6 | 室内ライト命令 |
| 7 | スポットライト命令（TA/TAB のみ使用） |

### 3.3 方向判定ロジック

1. 起動時、各マイコンはコネクタ識別マーカー（Number CH3）を読む。
2. `1.0` が来るコネクタを「前方コネクタ」、`0.0` を「後方コネクタ」と確定する。
3. TA は必ず前方コネクタが車体前方に接続されるよう設置する。
4. TAB は前方コネクタが車体後方（列車前方）に接続される。

## 4. マスター（運転権）システム

### 4.1 状態定義

| 状態 | 説明 |
|------|------|
| NO_MASTER | 誰もマスターを持っていない（初期状態） |
| TA_MASTER | TA（前部）がマスター |
| TAB_MASTER | TAB（後部）がマスター |

### 4.2 要件（EARS記法）

**REQ-M01** — マスター初期化
`システムは起動時にマスター状態を NO_MASTER に設定すること`

**REQ-M02** — マスター取得（TA）
`TA/一両車のマスターリクエスト入力がONになったとき、システムは当該車をマスターに設定し、TAB_MASTER 状態を解除する信号を後方へ伝播すること`

**REQ-M03** — マスター取得（TAB）
`TABのマスターリクエスト入力がONになったとき、システムは当該車をマスターに設定し、TA_MASTER 状態を解除する信号を前方へ伝播すること`

**REQ-M04** — 同時マスター排他
`マスター切り替え信号を受信したとき、現在マスターを持つ車両はマスターを解除すること（マスターは常に0または1）`

**REQ-M05** — Bool入力状態保持
`マスターが切り替わったとき、ドア・ライト等のBool出力状態は以前の状態を維持すること`

**REQ-M06** — マスター信号中継（TB）
`TB車がマスター要求信号を受信したとき、システムはその信号を反対側コネクタへ中継すること`

**REQ-M07** — 一両車（TA+TAB）
`同一マイコンにTA属性とTAB属性が両方設定されているとき、システムはそれを一両車として扱い、車間通信を行わないこと`

---

## 5. ドア制御

### 5.1 要件（EARS記法）

**REQ-D01** — 左右基準
`マスターが TA_MASTER のとき、システムは TA の前進方向を基準として左右を決定すること`

**REQ-D02** — 方向反転
`マスターが TAB_MASTER のとき、システムは TAB の前進方向（列車の逆方向）を基準として左右を決定し、TA基準の左右を反転して適用すること`

**REQ-D03** — ドア開閉命令伝播
`マスター車がドア開閉命令を出したとき、システムはその命令を全車両へ伝播すること`

**REQ-D04** — 非マスター車のドア適用
`TB/非マスター車がドア命令を受信したとき、マスター方向フラグに基づいてLeft/Right出力を正しくマッピングして適用すること`

---

## 6. 室内ライト・スポットライト制御

### 6.1 要件（EARS記法）

**REQ-L01** — 室内ライト伝播
`マスター車が室内ライト命令を出したとき、システムはその状態を全車両へ伝播すること`

**REQ-L02** — スポットライト適用範囲
`スポットライト命令を受信したとき、TA属性またはTAB属性を持つ車両のみスポットライト出力をONにすること`

---

## 7. スロットル・ブレーキ制御

### 7.1 要件（EARS記法）

**REQ-T01** — 命令伝播
`マスター車がスロットル・ブレーキ値を設定したとき、システムはその値を全車両へ伝播すること`

**REQ-T02** — 非マスター車の適用
`非マスター車がスロットル・ブレーキ命令を受信したとき、その値をそのまま自車出力チャンネルへ設定すること`

**REQ-T03** — マスター不在時
`NO_MASTER 状態の間、システムはスロットル=0・ブレーキ=1（フルブレーキ）を出力すること`

## 10. 実装計画

### フェーズ1: 単体車両動作（一両車）

- `controller/single_car.lua` — TA + TAB の機能を持つ単体車マイコン
- ローカル入力読み取り
- スロットル・ブレーキ・ドア・ライト出力
- マスター管理（一両なので車間通信なし）
- モニタ表示

### フェーズ2: 車間通信・マスター伝播

- `controller/car.lua` — TA/TB/TAB 汎用マイコン
- 車間通信プロトコル実装
- マスター要求・解除の伝播
- 命令中継ロジック

### フェーズ3: 統合テスト・調整

- 多両編成での動作確認
- ドア左右反転の検証
- エッジケース（一両車、2両編成など）

## 11. 依存関係・制約

| 制約 | 内容 |
|------|------|
| ファイル分割不可 | `require` 不可のため、1マイコン=1ファイルに全ロジックを記述 |
| 状態保持 | ティック間状態はグローバル変数またはアップバリューで保持 |
| 実行時間 | 1ティックあたり最大1000ms。重いループは避ける |
| ビークルリスポーン | リスポーン時に内部状態はリセットされる。状態復帰を考慮すること |
| マルチプレイ | ロジック入出力のみ同期。内部状態は各クライアントで独立 |
