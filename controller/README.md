# 列車コントローラー（1マイコン / 3スクリプト構成）

1個のマイコン内に Lua スクリプトを3つ置き、内部コンポジットで結線する。全車両が同一の3ファイルを搭載し、`Has Front Control` / `Has Back Control` プロパティのみで挙動が分岐する（単一ソース・データ駆動）。

| ファイル | 機能 | 対応設計 |
|---|---|---|
| `core.lua` | M1〜M6：入力・連結整列・マスター調停・GFF・指令パケット中継（**同一ティック完結のクリティカルパス**） | design.md §6.1–6.5 |
| `output.lua` | M7〜M9：走行・ドア・ライト出力写像 | design.md §6.6–6.8 |
| `display.lua` | M10：運転モニター 3×1 描画 | ui_design.md |

`output` / `display` は `core` の結果を消費するだけなので **core から1ティック遅延**するが、定常状態では不可視（系全体が元々マルチティック収束）。

## 分割理由（8KB 制約 / REQ-NFR-01）

各スクリプトは 8192 バイト制約を個別に持つ。統合版（約9.2KB）は1枚に収まらないため機能分割した。現状サイズ：core 8000 B / output 2557 B / display 2521 B（いずれも ≤8192）。

## 結線

### 1. core への入力（`core.in`）
- 前コネクタ由来 → 入力ch **1〜16**、後コネクタ由来 → 入力ch **17〜32**（物理コネクタ固定割り当て）。
- 運転手操作 → 空きch：N9=スロットル, N10=ブレーキ / B9=右ドア, B10=左ドア, B11=室内, B12=リクエスト, B13=非常, B14=バック, B15=スポット（全て push ボタン）。

### 2. core の出力（`core.out`）
1ノードに集約。内訳：
- **連結フレーム**：前=ch1〜16・後=ch17〜32 → 各コネクタへ（前後で別内容。ゲーム内オフセット配線で自車後フレーム出力(17-32)が隣の前入力(1-16)へ届くようにする＝design.md §12-R1）。
- **コマンド/表示バス**：下表のch。`core.out` を分岐して `output.in` と `display.in` へ内部配線（オフセットなしの素通し32ch）。

### 2.5 コネクタ⇄チャンネル オフセット配線（前後フレーム分離）★重要★

実機のコネクタは「**入力＝1ノードに合流**／**出力＝両コネクタ共通（同一内容）**」になっている。一方 `core` は **前コネクタ＝入力1〜16・後コネクタ＝入力17〜32**、**前フレーム＝出力1〜16・後フレーム＝出力17〜32（別内容）** を前提とする。このギャップを埋めるため、**後コネクタの配線に ±16 のチャンネルシフトを入れて前後フレームを分離**する。

> 重要：出力を両コネクタへ「同一内容」のまま流すと前後が区別できず、向き判定（整列/反転・進行方向）が破綻する。下記シフトにより**前コネクタと後コネクタが別フレームを運ぶ**ようにすること。

**入力側（コネクタ受信 → 単一入力ノード）**

| コネクタ | 配線上の受信ch | シフト | チップ入力ch |
|---|---|---|---|
| 前 | 1〜16 | なし（直結） | 1〜16 |
| 後 | 1〜16 | **+16** | 17〜32 |

**出力側（単一出力ノード → 各コネクタ送信）**

| コネクタ | チップ出力ch | シフト | 配線上の送信ch |
|---|---|---|---|
| 前 | 1〜16 | なし（直結） | 1〜16 |
| 後 | 17〜32 | **−16** | 1〜16 |

両コネクタとも**配線上は 1〜16 の16chフレーム**を送受信する（1コネクタ＝1フレーム分）。チップ内部でのみ前=1〜16／後=17〜32 に分かれる。これにより自車の後フレーム出力(17-32)が、隣車の前入力(1-16)へ届く（design.md §12-R1）。

```
            ┌──────────────── core チップ ────────────────┐
 前コネクタ ─┤ in 1-16  ◀───────────────────── 受信 1-16 (直結)
            │ out 1-16 ─────────────────────▶ 送信 1-16 (直結) ├─ 前コネクタ
            │                                              │
 後コネクタ ─┤ in 17-32 ◀── [ +16 シフト ] ◀── 受信 1-16     │
            │ out 17-32 ─▶ [ -16 シフト ] ──▶ 送信 1-16      ├─ 後コネクタ
            └──────────────────────────────────────────────┘
```

- **シフトの実現方法**：±16 のチャンネル移動は、ゲーム内の「コンポジット チャンネルシフター」コンポーネント、または `Composite Read Number/OnOff`＋`Composite Write Number/OnOff` で ch n→n±16 に付け替えて行う。ワークショップのチャンネルシフター MC を後コネクタの送受信2本に挟んでもよい。
- **前後の物理向きは不問**：どちら向きに連結しても、フレーム内の符号マーカー（前=+1／後=−1、§5.2）が「相手のどちらのコネクタから来たか」を伝えるため、整列/反転は自動判定される。シフトは自車の前後フレームを分離するだけで、相手の向きには依存しない。

# ユーザーガイド

> Lua の知識は不要。ゲーム内で本コントローラーを車両へ導入するための手順書。

## 1. 概要

連結した列車（編成）を、どの運転台からでも統一操作できる分散型コントローラー。各車両に同じマイコン1個（中に3スクリプト）を載せるだけで、向き・運転権・走行・ドア・ライトが編成全体で自動的に整合する。車両番号や向きの手動設定は不要。

## 2. 対象車種

全車両に**同じ3ファイル**（`core.lua` / `output.lua` / `display.lua`）を搭載する。車種は2つのプロパティ（§3）の組み合わせだけで自動的に決まる。

| 車種 | 運転台 | 動力 | 用途 |
|---|---|---|---|
| **TA**（前向き動力車） | 前のみ | あり | 前向き運転台の先頭/最後尾車 |
| **TAB**（後ろ向き動力車） | 後のみ | あり | 後ろ向き運転台の先頭/最後尾車 |
| **TB**（動力付き客車） | なし | あり | 中間車（運転台なし） |
| （ありえない組合せ） | 両方 | なし | ビルド誤り。安全のため中継のみ |

- 運転台付き車（TA / TAB）のみがマスター（運転権）になれる。編成の**端**に置く。
- 中間車はすべて TB。
- モニター（3×1）と前照灯/尾灯は TA / TAB のみ。

## 3. プロパティ設定

各マイコン内のプロパティで設定する。**この2つだけ**でよい（番号・向きの設定は不要）。

| ラベル | 型 | 説明 | デフォルト |
|---|---|---|---|
| Has Front Control | Bool | 前向き運転台を持つ車なら true | false |
| Has Back Control | Bool | 後ろ向き運転台を持つ車なら true | false |

| Has Front | Has Back | → 車種 |
|---|---|---|
| true | false | TA |
| false | true | TAB |
| false | false | TB |
| true | true | ありえない（中継のみ・走行不可） |

## 4. 入力チャンネル一覧

`core` スクリプトの入力は、**(A) コントローラー（操縦席）から受ける入力**と **(B) コネクタ（連結器）から受ける入力**の2系統が同じ入力ノードに同居する。両者でチャンネルが衝突しないよう割り当て済み。**配線するのは (A) のみ**で、(B) は連結器から自動で入る（§2.5 のオフセット配線で供給）。

### 4-A. コントローラー（操縦席）から受ける入力 ← **あなたが配線する**

運転手の操作レバー/ボタンをこれらのチャンネルへ繋ぐ。**Bool はすべて押しボタン（モーメンタリ）**：一度押すと ON、もう一度押すと OFF。

**Number 入力（TA / TAB のみ）**

| CH | 型 | 用途 | 値域 |
|---|---|---|---|
| 9 | Number | スロットルレバー | 0.0〜1.0 |
| 10 | Number | ブレーキスロットルレバー | 0.0〜1.0 |

**Bool 入力（押しボタン）**

| CH | 型 | 用途 | 対象車種 |
|---|---|---|---|
| 9 | Bool | 右ドアボタン | 全車 |
| 10 | Bool | 左ドアボタン | 全車 |
| 11 | Bool | 室内ライトボタン | 全車 |
| 12 | Bool | マスターリクエストボタン | TA / TAB |
| 13 | Bool | 非常/パーキングブレーキボタン | TA / TAB |
| 14 | Bool | バック（進行方向反転）ボタン | TA / TAB |
| 15 | Bool | スポットライトボタン | TA / TAB |

### 4-B. コネクタ（連結器）から受ける入力 ← **自動。手動配線しない**

隣車から連結器経由で届く信号。**前コネクタ由来＝ch1〜16／後コネクタ由来＝ch17〜32**（§2.5 のオフセット配線で振り分け）。ユーザーが触る必要はないが、**操縦入力をこれらのchへ繋がない**こと（連結判定・指令伝播が壊れる）。

**Number 入力**

| CH | 型 | 用途 |
|---|---|---|
| 1 / 17 | Number | 連結マーカー（接続検出＋前後識別、前/後） |
| 2 / 18 | Number | 認可スロットル（前/後） |
| 3 / 19 | Number | 認可ブレーキ（前/後） |

**Bool 入力**

| CH | 型 | 用途 |
|---|---|---|
| 1 / 17 | Bool | マスター在否（前/後） |
| 2 / 18 | Bool | 進行方向（送信車GFF、前/後） |
| 3 / 19 | Bool | 非常ブレーキ（前/後） |
| 4 / 20 | Bool | グローバル右ドア指令（前/後） |
| 5 / 21 | Bool | グローバル左ドア指令（前/後） |
| 6 / 22 | Bool | 室内ライト指令（前/後） |
| 7 / 23 | Bool | スポットライト指令（前/後） |
| 8 / 24 | Bool | マスター強制解放（前/後） |

> 「前/後」は、前コネクタ由来は左の番号、後コネクタ由来は右の番号を意味する（例：認可スロットルは前 ch2・後 ch18）。Number/Bool は別チャンネル空間。

## 5. 出力チャンネル一覧

`output` スクリプトの出力を車両機器へ繋ぐ。

**Number 出力**

| CH | 型 | 用途 | 値域 |
|---|---|---|---|
| 1 | Number | モーター（スロットル） | -1.0〜1.0 |
| 2 | Number | ブレーキ | 0.0〜1.0 |

**Bool 出力**

| CH | 型 | 用途 |
|---|---|---|
| 1 | Bool | 右ドア |
| 2 | Bool | 左ドア |
| 3 | Bool | 室内ライト |
| 4 | Bool | スポットライト（前端のみ点灯） |
| 5 | Bool | テールライト（後端のみ自動点灯） |

> スロットル出力は -1〜1 の符号付き。モーター/ギアボックスがこの符号で前後に回るよう接続する。

## 6. 接続手順

### マイコン内（3スクリプトの結線）

```
            [運転レバー/ボタン]
                   │
   ┌───────────────▼───────────────┐
   │            core               │
   │  (入力1-16=前 / 17-32=後)      │
   └───┬───────────┬──────────┬────┘
       │連結フレーム │バス      │バス
       ▼           ▼          ▼
  [前/後コネクタ] [output]   [display]
                   │            │
              [車両機器]   [3×1モニター]
```

- `core` の出力を分岐し、**連結フレーム**は前後コネクタへ、**コマンド/表示バス**は `output` と `display` の入力へ内部配線する（詳細は本書上部「結線」）。

### 車両間（コネクタ）

```
[前コネクタ] ── 隣車（前隣）
     │
  [自車マイコン]
     │
[後コネクタ] ── 隣車（後隣）
```

- 各車の前コネクタ・後コネクタを隣車のコネクタへ繋ぐだけ。**物理的な前後の向きは問わない**（逆向き連結でも自動整合）。

> ⚠️ **コネクタとチップの間にチャンネルオフセット配線が必須**。実機は「入力=1ノード合流／出力=両コネクタ共通」のため、後コネクタの送受信に **±16 のチャンネルシフト**を入れて前後フレームを分離する。詳細は本書上部「2.5 コネクタ⇄チャンネル オフセット配線」を参照。出力を両コネクタへ同一内容のまま流すと向き判定が壊れるので必ず分離すること。

## 7. 操作手順

すべて押しボタン操作（押すたびに切替）。**マスター（運転権）を取った運転台**の操作が編成全体に反映される。

- **マスターリクエストボタン（Bool IN CH12）**: 押すたびにマスター ON/OFF。ON で運転権取得＝この運転台で編成を操作可能になる。別の運転台で取得すると前のマスターは自動解放される（編成内マスターは常に0または1）。
- **スロットルレバー（Number IN CH9）**: マスター時、0→1 で編成が加速。進行方向はマスター運転台の向き（＋バック）で決まる。
- **ブレーキスロットルレバー（Number IN CH10）**: マスター時、常用ブレーキ 0→1。
- **非常/パーキングブレーキボタン（Bool IN CH13）**: 押すたびに ON/OFF。ON 中はブレーキ 100% 固定（スロットル無効）。
- **バックボタン（Bool IN CH14）**: 押すたびに進行方向反転。全車のスロットル符号・前照/尾灯が一括で入れ替わる。
- **右/左ドアボタン（Bool IN CH9 / CH10）**: 押すたびに開閉。マスター在線時はグローバル左右（編成全体で揃う）、マスター不在時は自車のみローカル操作。
- **室内ライトボタン（Bool IN CH11）**: 押すたびに ON/OFF。マスター在線時は編成全体、不在時は自車のみ。
- **スポットライトボタン（Bool IN CH15）**: 押すたびに ON/OFF。点灯するのは進行方向**前端**の運転台付き車のみ。

## 8. 注意事項

- **リスポーン直後はマスター不在**（走行不可・ブレーキ100%）。いずれかの運転台でリクエストを押して運転権を取得すること。数ティックで向き・進行方向が再確立する。
- **マスター不在時は走行できない**。ドア・室内ライトのみ各車ローカルで操作可。
- **運転台付き車を新たに連結すると、既存マスターは自動解放**される（過渡の荒ぶり防止）。連結後にリクエストを押し直すこと。
- **横取りされた運転台はリクエストが内部で OFF に戻る**。再取得にはもう一度押す（OFF→ON）必要がある。
- **入力 CH1〜32 は連結信号が占有**。運転レバー/ボタンは §4 の空きチャンネル（N9-10, B9-15）へ繋ぐこと。誤って占有chへ繋ぐと連結判定が壊れる。
- **Has Front / Has Back を両方 true** にすると「ありえない組合せ」となり走行しない（中継のみ）。運転台はどちらか一方だけ true にする。
- **後端が TB（運転台なし）の編成はテールライトが点灯しない**（テール出力を持つ車が後端に無いため。仕様上の許容）。
- 逆向きに連結した中間車も、ドア左右・モーター方向は自動で反転整合する。手動調整は不要。
